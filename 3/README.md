# AutoMarket — Лабораторная работа 3

Веб-приложение на **Flask**, которое одновременно использует **PostgreSQL** (автомобили) и **MongoDB** (отзывы).

---

## Структура проекта

```
carshop/
├── app.py                  # Основное приложение Flask
├── requirements.txt        # Зависимости Python
├── Dockerfile
├── docker-compose.yml      # Запуск всего стека одной командой
└── templates/
    ├── base.html           # Базовый шаблон (навигация, стили)
    ├── index.html          # Главная — список автомобилей
    ├── car_form.html       # Форма добавления автомобиля
    ├── car_detail.html     # Детальная страница + отзывы
    └── search.html         # Страница поиска
```

---

## Запуск через Docker (рекомендуется)

```bash
docker-compose up --build
```

Приложение будет доступно по адресу: http://localhost:5000

---

## Запуск вручную

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Запустить PostgreSQL и MongoDB

Убедитесь, что оба сервера запущены и доступны по стандартным портам (5432 / 27017).

### 3. Настроить переменные окружения (опционально)

```bash
export DATABASE_URL=postgresql://postgres:postgres@localhost:5432/carshop
export MONGO_URL=mongodb://localhost:27017/
export SECRET_KEY=my-secret-key
```

### 4. Инициализировать базу данных и запустить

```bash
export FLASK_APP=app.py
flask init-db       # создаёт таблицы и добавляет тестовые данные
flask run           # http://127.0.0.1:5000
```

## Маршруты

| Маршрут                   | Описание                                    |
|---------------------------|---------------------------------------------|
| `GET /`                   | Список всех автомобилей                     |
| `GET /cars/new`           | Форма добавления автомобиля                 |
| `POST /cars/new`          | Сохранение нового автомобиля в PostgreSQL   |
| `GET /cars/<id>`          | Детальная страница + отзывы из MongoDB      |
| `POST /cars/<id>`         | Добавление отзыва в MongoDB                 |
| `GET /search`             | Поиск по названию и минимальному рейтингу   |

## Модель данных

### PostgreSQL — таблица `cars`

| Поле          | Тип              | Описание                        |
|---------------|------------------|---------------------------------|
| `id`          | integer PK       | Первичный ключ                  |
| `name`        | varchar(20)      | Модель (обязательно)            |
| `brand`       | varchar(20)      | Марка (обязательно)             |
| `description` | text             | Описание                        |
| `price`       | decimal          | Цена (положительная)            |
| `stock`       | integer          | Количество на складе (≥ 0)      |

### MongoDB — коллекция `comments`

| Поле         | Тип      | Описание                          |
|--------------|----------|-----------------------------------|
| `_id`        | ObjectId | Автоматически                     |
| `product_id` | integer  | ID автомобиля из PostgreSQL       |
| `author`     | string   | Автор (по умолчанию «Аноним»)     |
| `text`       | string   | Текст отзыва (обязательно)        |
| `rating`     | integer  | Оценка 1–5 (обязательно)         |
| `created_at` | datetime | Дата и время (по умолчанию сейчас)|
